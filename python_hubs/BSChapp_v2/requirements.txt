## ğŸ”§ Troubleshooting

### HTML app not loading?

1. **Check file path:**
   
   ```
   html_apps/ai_storyboard.html  â† Must be exactly here
   ```
1. **Verify URL in browser:**
   
   ```
   http://localhost:8501/html_apps/ai_storyboard.html
   ```
   
   Should load directly (outside iframe)
1. **Check browser console** for errors (F12)

### Theme not switching?

- Ensure `subject_mode` is in `st.session_state`
- Portal page reads from main app session
- Refresh portal page after changing subject

### Signature not passing?

- Set signature on **main page first**
- Check session state: `st.session_state["signature"]`
- Portal reads this value and passes to URL

## ğŸ“ Technical Notes

### Why IFrame?

- **State persistence** - HTML app state survives page changes
- **Isolation** - HTML CSS doesnâ€™t affect Streamlit
- **Security** - Sandboxed execution
- **Flexibility** - Any HTML/JS/CSS works

### Glassy UI Implementation

Uses CSS `backdrop-filter: blur()` for frosted glass effect:

- Works on modern browsers (Safari, Chrome, Edge)
- Graceful degradation on older browsers
- Hardware-accelerated rendering

### Session State Sharing

Streamlit multipage apps share session state:

- Main app sets values
- Portal pages read values
- Works across navigation
- Persists during session
